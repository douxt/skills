# Rubric: 代码可维护性与风格

适用：所有代码改动（可维护性、命名、风格）
不适用：逻辑/边界/异常问题（见 correctness.md）、安全问题（见 security.md）
**硬性约束：本 rubric 仅关注可维护性与风格，不得评论逻辑正确性、安全性、性能。**

## 可读性
- [ ] 命名清晰，能反映变量/函数的用途
- [ ] 函数/类职责单一，不过大过深
- [ ] 无魔法数字（具名常量替代）

## 可维护性
- [ ] 无死代码（不可达分支、注释掉的代码、未使用变量/函数）
- [ ] 无硬编码敏感值（密钥、URL、数据库连接串）
- [ ] 重复代码已提取（3 次以上相同模式）
- [ ] 依赖方向合理（底层不依赖高层）

## 风格
- [ ] 符合项目现有代码风格（缩进、空格、引号）
- [ ] 注释解释「为什么」非「做了什么」（后者应被代码自解释）
