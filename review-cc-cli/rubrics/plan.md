# Rubric: 方案/计划评审

适用：设计方案、重构计划、技术决策文档
不适用：代码改动、配置变更

## 完整性
- [ ] 方案覆盖了所有已知的场景和约束
- [ ] 有明确的实施步骤或阶段划分
- [ ] 上下游依赖已识别并说明

## 可行性
- [ ] 方案在当前架构下可实施，不引入结构性冲突
- [ ] 资源/时间估算合理，有依据
- [ ] 技术选型有理由，不是随口选的

## 风险
- [ ] 已知风险已列出，有缓解措施
- [ ] 有回退方案或止损条件
- [ ] 兼容性影响（DB schema、API 版本、配置文件）已评估

## 可验证性
- [ ] 成功标准可衡量，不是模糊描述
- [ ] 有测试策略或验证方案
- [ ] **每个步骤有独立验证门禁**：该步骤改动完后，通过什么方式确认符合预期（编译/测试/lint/手动检查等），无验证手段的步骤应标注风险
