# Rubric: 配置/依赖评审

适用：CI 配置、依赖变更、环境变量、Dockerfile、compose、setting 文件
不适用：业务代码改动

## 依赖
- [ ] 新增依赖有明确用途，非"用用看"
- [ ] 版本号明确固定，无 `^`/`~` 浮动范围
- [ ] 无已知安全漏洞的依赖版本

## CI/CD
- [ ] CI 步骤顺序合理，无遗漏环节
- [ ] 敏感信息走环境变量/密钥管理，非硬编码
- [ ] 构建产物不做不必要的缓存或上传

## 环境
- [ ] 新增环境变量有默认值或文档说明
- [ ] 端口/路径不与现有服务冲突
- [ ] 容器资源限制合理
