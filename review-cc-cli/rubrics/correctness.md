# Rubric: 逻辑正确性与错误处理

适用：所有代码改动（逻辑、边界、异常、类型安全）
不适用：风格/命名/格式类问题（见 default.md）
**硬性约束：本 rubric 仅关注逻辑正确性，不得评论命名、格式、代码风格。**

## 逻辑正确性
- [ ] 逻辑符合预期，无明显的条件判断错误
- [ ] 边界条件已处理（空值、零、最大值、空集合、单元素）
- [ ] 循环/递归有终止条件，无无限循环风险
- [ ] 类型使用正确（无 `"1" == 1` 类隐式转换陷阱）
- [ ] 异步操作正确处理（Promise 未被忽略、竞态条件已防护）

## 错误处理
- [ ] 错误情况有处理，不是直接忽略或吞掉异常
- [ ] 外部调用（API、DB、文件）有容错保护
- [ ] 错误信息可定位（含上下文信息，非 "error" 或无信息抛出）
- [ ] 资源正确释放（文件句柄、连接、定时器）

## 状态与并发
- [ ] 状态转换完整（所有合法状态路径可达，非法路径被阻断）
- [ ] 共享数据访问有保护（锁/事务/原子操作）
- [ ] 幂等操作确实幂等（重复调用不产生副作用）
