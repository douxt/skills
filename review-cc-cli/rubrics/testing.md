# Rubric: 测试质量评审

适用：测试代码改动、新增测试、修复测试
不适用：不改动测试逻辑的代码

## 覆盖率
- [ ] 核心逻辑路径有测试覆盖
- [ ] 边界条件有测试（空值、异常值、极限值）
- [ ] 错误路径有测试（异常抛出、错误返回）

## 测试质量
- [ ] 测试验证行为而非实现细节（改实现不崩测试）
- [ ] 断言有明确的预期值，无空断言或注释掉的断言
- [ ] 测试可重复运行，不依赖外部状态或顺序

## 维护性
- [ ] 测试代码和生产代码一样有清理命名
- [ ] 无重复的测试样板（可用参数化或 helper 简化）
- [ ] 无脆弱的定时依赖或网络依赖
