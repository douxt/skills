# Rubric: 性能审查

适用：涉及数据库查询、大量数据处理、循环、热路径的代码
不适用：纯配置、文档、一次性的脚本

## 查询效率
- [ ] 无 N+1 查询（循环内调数据库或外部 API）
- [ ] 查询有索引，无全表扫描风险
- [ ] 批量操作用批量接口，非单条循环

## 内存与资源
- [ ] 无内存泄漏（事件监听器注销、连接关闭、文件句柄释放）
- [ ] 大数据集有分页或流式处理，非一次全量加载
- [ ] 无不必要的对象创建或深拷贝

## 热路径
- [ ] 循环内无高开销操作（IO、复杂计算、正则编译）
- [ ] 可缓存的结果有缓存（重复计算、重复查询）
- [ ] 无阻塞操作在异步上下文中运行
