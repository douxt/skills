# Rubric: 安全审计

适用：涉及认证、授权、输入处理、敏感数据、外部接口的代码
不适用：纯 UI/展示逻辑、不改动数据流的文档

## 输入验证
- [ ] 所有外部输入经过校验，无直接拼入查询或命令
- [ ] 路径遍历防护（`../`、绝对路径注入已处理）
- [ ] 文件包含防护（不可信路径做 allowlist）

## 认证与会话
- [ ] 敏感操作有权限检查，不只依赖前端隐藏
- [ ] Token/密钥不硬编码、不写日志、不传到客户端

## 数据安全
- [ ] SQL/NoSQL 查询使用参数化查询或 ORM，无字符串拼接
- [ ] 敏感数据传输使用 HTTPS，无明文传输
- [ ] 错误信息不泄露内部细节（栈信息、SQL 结构、文件路径）

## 命令执行
- [ ] 不直接拼接用户输入到 shell 命令
- [ ] 使用系统命令时有参数化或转义处理
