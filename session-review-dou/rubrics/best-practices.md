# 最佳实践速查卡

skill 诊断时对照此表，发现匹配模式直接给出对应命令。

## 循环升级

| 检测 | 升级到 | 命令模板 |
|------|--------|---------|
| 同命令手动重复 ≥5 次 | /goal | `/goal [任务] until [可计数的完成条件] stop after N` |
| 定时手动检查 ≥3 次 | /schedule | `/schedule every 1h: [检查命令]` |
| 多会话相同任务 | /loop | `/loop 1h /goal [任务] until [条件] stop after N` |
| 多步骤手动流程 | skill | 创建 SKILL.md，参考 session-review-dou 模板 |
| 每次新产出都手动检查 | /goal | `/goal 检查 [产出] 满足 [规则] until 全部通过 stop after 3 turns` |

## 错误预防

| 检测 | 措施 | 具体做法 |
|------|------|---------|
| AI 猜错字段/路径 | 反向复述检查点 | `动代码前先用自己话写出对每个关键字段的理解，等我确认` |
| AI 声称完成但无证据 | 完成标准 | `until [机器可检查条件]` + 要求输出验证日志 |
| 同一错误 ≥2 次 | CLAUDE.md lessons | `- [xN] Mistake: ... Correct: ... Why: ...` |
| 开工前未核实数据 | 基线核查 | `任何数量型事实亲自执行命令确认，标注快照时间` |

## 协作升级

| 检测 | 措施 | 具体做法 |
|------|------|---------|
| 方案在多个方向间振荡 | 前提对齐 | `写方案前先确认 3-5 条不可动摇的原则` |
| 子代理编造事实 | 双重验证 | `子Agent启动时先验证父会话前提条件` |
| 过度工程 | 约束前置 | `确认用户数/数据量/频率/容忍度后再设计` |

## /goal 三要素公式

```bash
/goal [任务描述]
until [可数量化的结束状态]
stop after [N]
```

例：
```bash
/goal 检查最近3天总结的frontmatter完整性
until 全部含video_id+title+date
stop after 3 turns
```

## 四问定位瓶颈

```
① AI 第一次回复就走偏？→ 意图对齐瓶颈 → 加反向复述
② AI 说修好了但没修好？→ 完成验证瓶颈 → 加机器可检查标准
③ 同样的错又犯了？→ 错误固化瓶颈 → 写 lessons
④ 你重复做同样准备？→ 循环升级机会 → 上 /goal 或 /schedule
```
