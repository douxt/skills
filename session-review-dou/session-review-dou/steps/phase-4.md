# 阶段 4：输出与归档

## 终端报告

输出完整报告（六维诊断表 + 建议列表 + 升级路径）。
总结一句话：当前处于哪个循环阶段（Turn-based/Goal-based/Time-based/Proactive），下一步应该做什么。

格式：
```
# 会话尸检报告 — <日期>

## 总览
分析 N 个会话，发现 M 个问题，其中 P0 级 K 个。

（阶段 2 诊断表格）

（阶段 3 建议列表）

## 升级状态
当前：Turn-based（手动触发所有任务）
下一步：Goal-based — 流水线扫描已可量化完成标准，升级为 /goal

## 规则生效跟踪
| 上次规则 | 状态 |
|----------|------|
| 开工前前提确认 | ✅ 本次同类信号消失 |
| （如无上次归档则显示"首次运行，无历史对比"）|
```

## 归档

所有报告（含"✅ 无可写规则"的空报告）写入 `reviews/YYYY-MM-DD.md`。
格式：上述终端报告的完整 Markdown。

## 写入确认

对 ≥3 次的发现，逐一询问用户：
```
是否将以下规则写入 CLAUDE.md？
1. 开工前基线核查 — 来源 3 个会话
  [y] 写入  [s] 跳过  [e] 编辑后再写
```
用户确认后才执行写入。不自动写入。

（阶段 4 结束。会话尸检完成。）
